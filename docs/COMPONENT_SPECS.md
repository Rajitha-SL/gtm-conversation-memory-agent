# Component Specifications & JSON Payload Specs

This document defines the interface schemas, header protocols, and transactional lifecycle contracts for the three phases of the GTM Context Engine Pipeline.

---

## 1. Ingestion Phase (Call Transcript Webhook)
Receives raw call metadata and meeting transcripts from Gong or simulators, validates authenticity, stages the database record, and queues the job for analysis.

### 1.1 Endpoint Configuration
- **Path**: `POST /api/v1/webhooks/gong`
- **Headers**:
  - `Content-Type: application/json`
  - `x-gong-signature: <hmac-sha256-signature>` (Used for authenticating payloads in production)

### 1.2 Request Schema (JSON)
```json
{
  "callId": "call_94b1774_abc",
  "title": "AWS Migration Objection Sync",
  "duration": 1800,
  "participants": [
    "devops_lead@targetcompany.com",
    "sales_engineer@ourcompany.com"
  ],
  "rawTranscript": "DevOps Lead: We are heavily locked into AWS. Moving off would be a huge operational cost blocker...",
  "recordedAt": "2026-07-17T08:27:00Z"
}
```

### 1.3 Response Contracts
- **202 Accepted (Enqueued)**:
  ```json
  {
    "status": "success",
    "message": "Transcript payload safely enqueued",
    "jobId": "50d2fbe0-6206-4d2d-b795-a74d7bff4ffa"
  }
  ```
- **400 Bad Request (Validation Failure)**:
  ```json
  {
    "statusCode": 400,
    "error": "Bad Request",
    "message": "body must have required property 'rawTranscript'"
  }
  ```
- **429 Too Many Requests (Rate Limited)**:
  ```json
  {
    "statusCode": 429,
    "error": "Too Many Requests",
    "message": "Rate limit exceeded, retry in 52 seconds"
  }
  ```

## 2. Processing & Memory Phase (LLM Processing & Embedding Ingestion)
Asynchronously processes the transcript queue, generates a structured summary using Gemini, commits summaries to PostgreSQL, and indexes them for vector search.

### 2.1 Database Record Staging Contract (`CallSummary` Model)
| Column Name | Prisma Type | Constraints | Purpose |
|-------------|-------------|-------------|---------|
| `id` | `String` | `@id @default(uuid())` | Unique job entry identifier |
| `callId` | `String` | `@unique` | Original external source key |
| `rawTranscript` | `String` | | Source text loaded during Ingestion |
| `aiAnalysisPass`| `String?` | `@default("")` | Summarized insight payload generated by Gemini |
| `status` | `String` | `@default("PROCESSING")` | Ingestion status (`PROCESSING`, `COMPLETED`, `FAULTED`) |
| `outboundTriggered`| `Boolean` | `@default(false)` | Tracks if outbound webhook has been dispatched |

### 2.2 Gemini Model Structured Output Specification (JSON)
The background worker expects the generative model to output a structured JSON analysis report matching the schema below:
```json
{
  "summary": "The DevOps Lead expressed strong concerns regarding the operational overhead of migrating their existing AWS instances due to cost locks.",
  "keyTakeaways": [
    "AWS remains the dominant infrastructure provider for the prospect.",
    "Cost migration objection is currently blocking deal progression."
  ],
  "sentiment": "NEUTRAL_WITH_OBJECTIONS",
  "buyingSignals": [
    "Interested in performance parity options",
    "Acknowledge inefficiencies in current AWS setup"
  ],
  "actionItems": [
    {
      "task": "Draft a TCO comparison model comparing our pricing to AWS",
      "owner": "sales_engineer@ourcompany.com"
    }
  ]
}
```

## 3. Activation Phase (Clay Target Enrichment & Outreach Automation)
Injects processed growth insights into automated GTM campaigns (e.g. Clay) to trigger target account enrichment and outreach workflows.

### 3.1 Endpoint Configuration
- **Path**: Configured via `CLAY_WEBHOOK_URL` in environment.
- **Headers**:
  - `Content-Type: application/json`

### 3.2 Outbound Webhook Schema (JSON)
```json
{
  "contactEmail": "devops_lead@targetcompany.com",
  "companyName": "TargetCompany Inc.",
  "gtmInsights": "AWS infrastructure migration cost objection identified during sync.",
  "buyingIntentScore": 65,
  "suggestedOutreachSequence": "objection-handling-aws-tco"
}
```

### 3.3 Callback & Verification Contract
Upon successful delivery to the Clay gateway (indicated by a `2xx` HTTP response status), the background worker updates the database flag `outboundTriggered` to `true` to lock the transactional event lifecycle.
